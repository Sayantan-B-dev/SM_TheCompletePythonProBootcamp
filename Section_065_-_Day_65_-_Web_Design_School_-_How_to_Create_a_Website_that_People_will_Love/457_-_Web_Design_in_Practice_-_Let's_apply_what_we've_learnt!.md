# III. Advanced UI Components to Learn

* Design Systems (Reusable UI foundation library)
* Component Variants (State-driven style variations)
* Design Tokens (Centralized style variables like color and spacing)
* Auto Layout (Responsive container logic in Figma)
* Grid Systems (Structured column alignment system)
* Motion Microinteractions (Feedback-driven animation)
* Accessibility Standards (Inclusive design compliance rules)
* Responsive Breakpoints (Device-based layout changes)

# IV. Step-by-Step: Installing and Setting Up Figma

## Step 1: Download Figma

1. Visit figma.com
2. Create an account
3. Download desktop application
4. Install normally like standard software

## Step 2: Understand Interface

* Frame (Artboard container for screens)
* Component (Reusable UI element)
* Auto Layout (Responsive stacking system)
* Constraints (Responsive positioning rules)
* Styles (Reusable typography or color settings)

---

## Step 3: Creating Your First UI Layout

1. Create Frame → Choose Desktop or Mobile size
2. Apply Grid → 12-column grid for desktop standard
3. Add Header Section
4. Insert Navigation text
5. Create Hero Section with heading and CTA
6. Maintain consistent spacing (8px spacing system recommended)

# V. Designing in Canva

Canva is simpler and more template-driven than Figma.

## Canva Workflow

1. Choose Website or Social Post template
2. Modify typography and colors
3. Adjust spacing manually
4. Export as PNG or shareable link

Limitations:

* No advanced component logic
* Limited responsive controls
* Not ideal for complex UX systems

Use Canva primarily for visual marketing graphics rather than product UI systems.

# VI. Design Principles to Write on Paper First

Before opening digital tools, define foundational thinking.

## Write These Sections on Paper

* Target Audience (Who the product serves and their cognitive expectations)
* Problem Statement (Specific issue being solved)
* Core Goal (Primary user outcome desired)
* User Flow (Step-by-step journey from entry to completion)
* Content Hierarchy (Which information appears first and why)
* Feature Priority (Must-have vs optional elements)
* Emotional Tone (Professional, playful, premium, minimal)
* Platform Context (Desktop-first or mobile-first strategy)

This prevents random design decisions.

---

# VII. Wireframing on Paper First

Draw low-fidelity structure before styling.

Include:

* Header
* Content sections
* CTA placement
* Footer

Avoid color at this stage. Focus purely on structure and hierarchy.

# VIII. Step-by-Step Full UI / UX Process

## 1. Research Phase

* Market research
* Competitor analysis
* User persona definition
* Behavioral pattern study

## 2. Strategy Phase

* Define design goals
* Outline conversion points
* Plan hierarchy system

## 3. Wireframe Phase

* Low-fidelity grayscale layouts
* Structure-only approach
* Validate content grouping

## 4. Visual Design Phase

* Apply typography system
* Define color palette
* Add imagery
* Maintain contrast rules

## 5. Prototype Phase

* Link screens in Figma
* Add microinteractions
* Test navigation flow

## 6. Testing Phase

* Usability testing
* Identify confusion points
* Improve clarity

## 7. Development Phase

* Export assets
* Provide developer handoff specs
* Maintain design token documentation

# XI. Common Mistakes to Avoid

* Starting design without defined problem
* Copying visuals without understanding structure
* Using too many fonts or colors
* Ignoring accessibility
* Designing without grid alignment
* Overloading content into single screen
* Skipping user testing

# XII. Professional Growth Path

- Beginner → Recreate existing designs
- Intermediate → Build original structured projects
- Advanced → Create full design systems
- Expert → Lead UX research and scalable architecture planning
